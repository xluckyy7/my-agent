## 0. The story in one paragraph

Started with a single API call (v0.0) and ended with a full agent harness
covering REPL + Web UI + 7 tools + MCP ecosystem + sub-agents + cross-session
memory + auto context compaction + a Claude-style hook framework + a
langfuse observability plugin. Each iteration shipped a tagged release,
each release has a retro doc capturing decisions and known limits. No
agent frameworks (no LangChain / no Agent SDK / no OpenAI Assistants) —
only `openai` SDK as the API wire, `mcp` SDK for protocol, `langfuse` for
observability, `httpx` / `trafilatura` / `tiktoken` as utility libs. Everything
else is hand-written.

## 1. What it can do (by capability)

### 1.1 Conversation

| Capability | How to use | Lives in |
|---|---|---|
| One-shot CLI question | `python -m my_agent "..."` | `cli/main.py` |
| Multi-turn REPL | `python -m my_agent` (no args) | `cli/repl.py` |
| Web UI (sidebar + chat) | `python -m my_agent.web` → browser http://localhost:8000 | `web/app.py` |
| Save / load sessions | `/save path.json` `/load path.json` | `agent/conversation.py` |
| Reset history | `/reset` (keeps system prompt) | `cli/repl.py` |

### 1.2 Built-in tools (always available)

| Tool | What | File |
|---|---|---|
| `read_file` | Read UTF-8 text file | `tools/files.py` |
| `write_file` | Write file, auto-mkdir parents, overwrites | `tools/files.py` |
| `run_bash` | Shell command with 30s timeout, captures stdout/stderr/exit | `tools/shell.py` |
| `web_fetch` | HTTP GET + trafilatura article extraction, max 8000 chars | `tools/web.py` |
| `web_search` | Tavily search (requires `TAVILY_API_KEY`; auto-registered when set) | `tools/web.py` |
| `remember` | Append a line to `~/.my-agent/memory/MEMORY.md` (cross-session) | `tools/memory_tool.py` |
| `task` | Spawn focused sub-agent for a self-contained subtask | `tools/task_tool.py` |

### 1.3 Streaming + UX

- Text streams token-by-token (uses OpenAI `stream=True`, normalized through `LLMClient.stream`)
- Tool indicators render inline: `▸ tool_name {args preview}` → `✓ 0.12s {result preview}` / `✗ error`
- ANSI colors auto-disable when stdout is not a TTY (pipe/redirect safe)
- `readline` enabled — Chinese / emoji backspace works correctly, arrow-key history available
- Web UI: vanilla JS single page (no React), ChatGPT-style sidebar with per-session titles auto-derived from first user message
- **Web sessions are persisted** (v1.1.1) to `~/.my-agent/web-sessions/<id>.json` — survives `uvicorn` restart, one file per session, hydrated on app construction

### 1.4 Slash commands (REPL)

| Command | Aliases | Purpose |
|---|---|---|
| `/help` | `/?` | List commands |
| `/quit` | `/q` `/exit` | Exit |
| `/reset` | | Clear history (keep system) |
| `/save <path>` | | Save conversation JSON |
| `/load <path>` | | Replace conversation from JSON |
| `/tokens` | | Show token count vs budget + msg/turn count |
| `/compact` | | Manually trigger compaction |
| `/memory [clear]` | | Show or clear user-level memory |
| `/mcp` | | List active MCP servers + their tools |

Keyboard:
- ctrl-d → exit
- ctrl-c at prompt → double-tap (within 2s) to exit / single-tap clears input
- ctrl-c during turn → interrupt turn, REPL continues

### 1.5 Context management (Iter 5)

- `ContextManager` runs before every `send()`. Triggers when conv ≥ 80% of `CONTEXT_BUDGET` (default 8000 tokens).
- Strategy: sliding window — keep `system` + last `KEEP_RECENT_TURNS` (default 4) user-turn-blocks intact;
  middle block summarized by LLM into a single `user`-role marker message prefixed `[CONVERSATION SUMMARY]`.
- Splits at user-message boundaries so `assistant.tool_calls ↔ tool` pairing is never broken.
- Configurable via env: `CONTEXT_BUDGET`, `KEEP_RECENT_TURNS`.

### 1.6 Cross-session memory (Iter 7)

- Two files auto-loaded into system prompt at startup:
  - `./AGENT.md` — project-level, git-tracked, human-edited
  - `~/.my-agent/memory/MEMORY.md` — user-level, LLM writes via `remember` tool
- `compose_system_prompt(base, project, user)` order: base → project → user
- Cross-process: new `python -m my_agent` session reads what previous wrote

### 1.7 Sub-agent (Iter 9)

- `task("...")` spawns an isolated sub-agent with same tools + own Conversation
- Intermediate tool calls inside sub-agent **hidden from parent**
- Parent only sees the sub-agent's final text summary
- Depth cap = 2 (sub-sub-agents have no `task` tool, so no infinite recursion)

### 1.8 MCP client (Iter 8)

- Configure `~/.my-agent/mcp.json` (same schema as Claude Desktop)
- On startup: spawn server subprocesses, fetch tool schemas, register as `<server>__<tool>` namespaced tools
- stdio transport only (no HTTP/SSE transport yet)
- A misconfigured server logs to stderr and is skipped — never blocks startup

### 1.9 Hook system (Iter 11)

- Configure `~/.my-agent/hooks.json` — schema mirrors Claude Code
- 7 events: `SessionStart` / `UserPromptSubmit` / `PreModelCall` / `PostModelCall` / `PreToolUse` / `PostToolUse` / `Stop`
- 2 hook types:
  - `command` — shell script, gets event JSON on stdin
  - `python` — `module:function`, receives `HookEvent` dataclass directly
- `matcher` regex filters hooks by subject (tool name / model name)
- **Never raises** — broken hook logs to stderr, agent continues

### 1.10 Built-in plugin: Langfuse observability

- `src/my_agent/plugins/langfuse_plugin.py` — 6 hook entry points
- Auto-creates per-session traces with nested spans:
  - `turn` (UserPromptSubmit → Stop)
    - `llm_call` × N (one per round) — captures model, messages, finish_reason, usage
    - `tool:<name>` × M — captures arguments, output, error flag
- Optional install: `pip install -e ".[langfuse]"`
- Optional env: `LANGFUSE_PUBLIC_KEY` / `LANGFUSE_SECRET_KEY` / `LANGFUSE_HOST`
- Without keys → graceful noop (plugin disabled, agent unaffected)
- Wire up: copy `docs/notes/example-hooks.json` → `~/.my-agent/hooks.json`

## 2. Architecture summary

```
   CLI / REPL  ─┐                        ┌─ FastAPI + SSE (web)
                │                        │
                └─→ AgentLoop ←──────────┘
                    .run_turn_stream
                    │
                    │  HookManager.fire  → command / python plugins
                    │  ContextManager.maybe_compact
                    │
       ┌────────────┼─────────────────┐
       ▼            ▼                 ▼
  Conversation   ToolRegistry      LLMClient
  + validate()   (built-in +       (openai SDK)
  + save/load     MCP +
   from JSON)     task)
                    │
                    └→ Tool.fn(args) → str
```

Three orthogonal cross-cuts woven through:
- **Hooks** — observers on event boundaries (Iter 11)
- **Context compaction** — auto-triggered before each LLMClient.send (Iter 5)
- **Memory injection** — once at startup into system prompt (Iter 7)

## 6. Known limitations (consolidated)

Pulled from each iter's retro. Sorted by user-impact priority.

### 🔴 High-impact / will hit in real use

1. **MCP every tool call respawns server** (~3-13s latency per call). v0.8.x should add persistent connection via background asyncio thread. See [`iter-8-retro`](iter-8-retro.md).
2. **`remember` is append-only** — no dedup, no update, no consolidate. After weeks of use MEMORY.md grows and may contain contradictions. v0.7.1 plan: add `update_memory(match, new)` and `forget(match)` tools + `/memory consolidate` REPL command. See [`iter-7-retro`](iter-7-retro.md).
3. **Compaction fails on huge single messages** (50KB tool result, big paste). `maybe_compact` returns False because keep-window covers it; conv stays over budget → next send may 400. Fix: iterative compaction + tool-result truncation. See [`iter-5-retro`](iter-5-retro.md).

### 🟡 Medium-impact / nice-to-have

4. **Hooks are observe-only** — cannot block / mutate tool calls. Claude Code's hook can return `{"block": true}`; we don't parse hook output yet.
5. **Web UI not Markdown-rendered** — model output shown as plain text. Code blocks / lists / headings appear literally.
6. **MCP only stdio + tools** — no HTTP/SSE transport, no resources / prompts primitives.
7. **No cost tracking** — usage_details captured in langfuse but no in-CLI `/cost` command.
8. ~~Web sessions in-memory only~~ — **fixed (v1.1.1):** sessions now mirror to `~/.my-agent/web-sessions/<id>.json` on every mutation and hydrate on startup. Survives uvicorn restart.

### 🟢 Low-impact / nice polish

9. **No Markdown rendering / code highlighting** in REPL or web.
10. **No dark mode** in web UI.
11. **No web auth** — local-only assumption.
12. **No `/memory reload`** — must restart agent to pick up MEMORY.md edits made during the session.
13. **Hook config typo doesn't show line number** — generic JSON error.

## 8. Non-goals (intentional)

- **Not a general framework** — designed for single-developer self-use, not for shipping to teams of agents
- **No multi-tenant auth** — local-only by design
- **No agent marketplace / plugin registry** — plugins are Python modules, install via pip / git clone
- **No GUI editor for memory / hooks** — plain markdown + JSON, edit in your text editor
- **No fine-tuning / model training** — purely an inference-time agent
- **No agent orchestration DSL** — task tool is the only multi-agent primitive

## 9. Where to next

If you want to **extend this project:**

- **v1.1.x quick wins** (≤1 day each):
  - MCP persistent connection (10× speedup)
  - `remember` update/forget tools
  - `/cost` command using langfuse usage data
  - Markdown rendering in web UI (marked.js, ~50 lines)
  - Hook block semantics (parse stdout JSON for `{block: true}`)

- **v1.2 medium projects** (~1 week each):
  - ~~File-backed sessions for web~~ — done in v1.1.1 (JSON file per session)
  - Compaction iterative + tool-result truncation
  - Per-memory file split (Claude Code style) + index
  - Web auth (single password or magic link)

If you want to **stop building and use it:**

- Run `python -m my_agent.web`, open browser, work
- Periodically `cat ~/.my-agent/memory/MEMORY.md` and tidy it
- Watch langfuse dashboard for cost / latency patterns
- Write iter-N retros' "我的补充" sections when interesting things happen

If you want to **apply lessons elsewhere:**

- The `Conversation.validate()` invariant pattern transfers to any protocol-strict integration
- The "hooks fire but never raise" pattern transfers to any observability/extensibility layer
- The "build_X(home, client)" factory pattern works whenever you need testable per-process state
- The retro-per-iter discipline scales to any multi-week learning project
